## 🔄 How Update Checking Works

### **1. Automatic Check on App Launch**

When the app starts, `AppController` automatically checks for updates:

```dart
// In AppController.onInit()
@override
void onInit() async {
  super.onInit();
  PackageInfo packageInfo = await PackageInfo.fromPlatform();
  currentVersion.value = packageInfo.version; // e.g., "1.2.50"
  
  if (!kIsWeb) {
    await checkLatestVersion(); // Checks GitHub for latest release
  }
}
```

### **2. Manual Check in About Page**

Users can manually check for updates:

**Location**: Profile → About → "Check for Updates"

```dart
// In about_page.dart (line 200-205)
_MenuRow(
  icon: Icons.system_update_outlined,
  iconColor: Colors.green,
  label: 'Check for Updates',
  textColor: textColor,
  onTap: () => appController.checkForUpdate(),
),
```

## 🔍 Technical Implementation

### **Current Version Detection**

The app reads its version from `pubspec.yaml`:

```yaml
# pubspec.yaml
version: 1.2.50  # ← This is your current version
```

This translates to:
- **versionName**: 1.2.50 (shown to users)
- **versionCode**: Auto-generated by Flutter

### **Latest Version Detection**

The app checks GitHub Releases API:

```dart
// Fetches from: https://api.github.com/repos/nabeelkts/drivemate/releases/latest
final response = await http.get(
  Uri.parse('https://api.github.com/repos/$repositoryOwner/$repositoryName/releases/latest'),
);

// Extracts tag_name (e.g., "v1.2.51")
final tagName = data['tag_name'];
```

### **Version Comparison**

```dart
String current = _extractVersion("1.2.50");  // "1.2.50"
String latest = _extractVersion("v1.2.51");  // "1.2.51"

if (current != latest) {
  // Show update dialog
} else {
  // Show "Up to Date" message
}
```

### **For Users:**

1. Open the app
2. Go to **Profile** tab
3. Tap **About**
4. Tap **"Check for Updates"**
5. If update available → Follow prompts to update
6. If up to date → See confirmation message

### **For Developers (You):**

#### **To Release an Update:**

1. **Update Version in Code**
   
   Edit `pubspec.yaml`:
   ```yaml
   version: 1.2.51  # Increment from 1.2.50
   ```

2. **Build Release**
   ```bash
   flutter build appbundle --release
   ```

3. **Upload to Play Store** (or create GitHub release)

4. **Create GitHub Release** (for in-app checking):
   - Go to: https://github.com/nabeelkts/drivemate/releases
   - Click "Create new release"
   - Tag version: `v1.2.51`
   - Title: "Version 1.2.51"
   - Description: Add release notes
   - Publish release

5. **Users will now see update prompt** when they open the app or manually check

## 🔧 Customization Options

### **Option 1: Force Update (Block App Usage)**

Currently, users can skip updates. To force updates:

Edit the dialog in `app_controller.dart`:

```dart
WillPopScope(
  onWillPop: () async {
    return false; // Prevent back button
  },
  // Remove "LATER" button
  // Keep only "UPDATE" button
)
```

### **Option 2: Use Firebase Remote Config**

Instead of GitHub, use Firebase to control versions:

1. Add `firebase_remote_config` package
2. Store `latest_version` in Remote Config
3. Check against Remote Config instead of GitHub

### **Option 3: Use Play Store In-App Updates API**

For Android only, use Google's official API:

```dart
import 'package:in_app_update/in_app_update.dart';

// Check for update
var info = await InAppUpdate.checkForUpdate();
if (info.updateAvailability == UpdateAvailability.updateAvailable) {
  // Start update flow
  InAppUpdate.performUpdate();
}
```

## 🚨 Common Issues & Solutions

### **Issue 1: "Failed to fetch GitHub release info"**

**Cause**: GitHub API rate limit or network error

**Solution**:
- Wait a few minutes (rate limits reset hourly)
- Check internet connection
- Verify GitHub repository exists and is public

### **Issue 2: Version comparison not working**

**Cause**: Version format mismatch

**Solution**:
- Ensure GitHub tag format: `v1.2.50` or `1.2.50`
- Ensure `pubspec.yaml` format: `1.2.50`
- The regex `_extractVersion()` handles both formats

### **Issue 3: Dialog doesn't appear**

**Cause**: `_updateDialogOpen` flag stuck

**Solution**:
- Restart app
- Or add timeout to reset flag:
  ```dart
  Future.delayed(Duration(seconds: 30), () {
    _updateDialogOpen = false;
  });
  ```
